## Все команды

### /start — Запуск бота

**Что делает:** Приветствует пользователя и показывает список всех возможностей.

**Когда использовать:** При первом запуске бота или чтобы вернуться к началу.

**Пример ответа:**
```
Здравствуйте, Ibrat! 👋

Я RAOS Support Bot — бот технической поддержки.

Я могу помочь вам с:
📋 Частые вопросы и ответы
📊 Проверка статуса системы
🎫 Создание заявки в поддержку

Команды:
/faq — Вопросы и ответы
/status — Статус системы
/ticket — Создать заявку
/help — Все команды

Или просто напишите свой вопрос — я постараюсь ответить! 💬
```

### /faq — Частые вопросы и ответы

**Что делает:** Показывает интерактивное меню с 8 категориями. Нажимаете на категорию — видите список вопросов. Нажимаете на вопрос — получаете ответ.

**Когда использовать:** Когда нужна помощь по работе с RAOS.

**Категории:**

| Кнопка | Что внутри |
|--------|-----------|
| Вход | Не могу войти, как сменить пароль |
| Товары | Как добавить товар, штрих-код, изменить цену |
| Продажи | Как продать, как сделать возврат |
| Оплата | Способы оплаты, как оформить насию (долг) |
| Отчёты | Отчёт по продажам, как закрыть смену |
| Синхронизация | Что делать без интернета, программа не работает |
| Печать | Как напечатать чек, как напечатать этикетку |
| Общее | Цена RAOS, как связаться с поддержкой, филиалы, сотрудники |

**Как работает:**
1. Отправьте `/faq`
2. Появятся кнопки с категориями (по 2 в ряду)
3. Нажмите на нужную категорию
4. Появится список вопросов
5. Нажмите на вопрос — получите ответ
6. Если ответ не помог — кнопка "Не помогло — создать тикет"
7. Кнопка "Назад" возвращает к предыдущему уровню

### /status — Статус системы

**Что делает:** Проверяет все сервисы RAOS в реальном времени и показывает подробный отчёт.

**Когда использовать:** Когда что-то не работает и нужно понять в чём проблема.

**Что проверяет:**

| Сервис | URL | Что это |
|--------|-----|---------|
| API | api.raos.uz | Бэкенд (база данных, авторизация, товары) |
| API Ready | api.raos.uz | Готовность БД и Redis |
| Web Admin | app.raos.uz | Панель управления (веб-интерфейс) |
| Landing | raos.uz | Главный сайт |

**Значения статусов:**

| Иконка | Значение |
|--------|----------|
| 🟢 | Работает нормально |
| 🟡 | Работает медленно (>3 секунды) |
| 🔴 | Не работает |

**Пример ответа (всё работает):**
```
📊 Статус сервисов RAOS

🟢 API — OK (145ms)
🟢 API Ready — OK (230ms)
🟢 Web Admin — OK (890ms)
🟢 Landing — OK (320ms)

🕐 Проверка: 15:30:00
✅ Все сервисы работают нормально!
```

**Пример ответа (есть проблема):**
```
📊 Статус сервисов RAOS

🟢 API — OK (145ms)
🟢 API Ready — OK (230ms)
🔴 Web Admin — НЕ РАБОТАЕТ
   Ошибка: TIMEOUT — сервер не ответил за 10 секунд
   URL: https://app.raos.uz
   Сбоев подряд: 15

   🔍 Диагноз: Таймаут соединения
   Причина: Сервис не отвечает — возможно крашнулся или Railway перезапускает
   Как исправить:
   1. railway service link <name> && railway logs --lines 50
   2. Ищи: OOM, MODULE_NOT_FOUND, ENOENT
   3. Если OOM: NODE_OPTIONS=--max-old-space-size=512
   4. Redeploy: git commit --allow-empty && git push origin main

🟢 Landing — OK (320ms)

🕐 Проверка: 15:30:00
⚠️ Есть проблемы. Если нужна помощь — /ticket
```

**Какие ошибки распознаёт:**

| Ошибка | Что значит | Причина |
|--------|-----------|---------|
| TIMEOUT | Сервер молчит | Крашнулся или перезапускается |
| NXDOMAIN | Домен не найден | DNS запись удалена из Cloudflare |
| SSL | Ошибка сертификата | Railway не выпустил SSL |
| HTTP 502 | Bad Gateway | Приложение падает при старте |
| HTTP 503 | Service Unavailable | Идёт деплой |
| HTTP 500 | Internal Server Error | Ошибка в коде |
| ECONNREFUSED | Соединение отклонено | Порт не слушает |
| ECONNRESET | Соединение сброшено | Сервер оборвал соединение |

### /ticket — Создать заявку в поддержку

**Что делает:** Создаёт заявку и отправляет уведомление команде.

**Когда использовать:** Когда FAQ и /status не помогли, нужна помощь от человека.

**Как использовать:**

Вариант 1 — с текстом:
```
/ticket Не могу войти в систему после смены пароля
```

Вариант 2 — без текста (покажет инструкцию):
```
/ticket
```

**Пример ответа:**
```
✅ Заявка создана!

🎫 Номер: T-M3KF5P
📝 Описание: Не могу войти в систему после смены пароля
📅 Дата: 04.06.2026, 16:30:00
📊 Статус: Открыта

Наша команда рассмотрит вашу заявку в ближайшее время.
```

**Что происходит после создания:**
- Заявка сохраняется в боте
- Уведомление отправляется команде поддержки
- Команда видит: кто написал, username, chat ID, текст проблемы

### /mytickets — Мои заявки

**Что делает:** Показывает список ваших последних заявок (до 5 штук).

**Когда использовать:** Чтобы проверить статус своих заявок.

**Пример ответа:**
```
📋 Ваши заявки:

1. 🟡 Не могу войти в систему после смены пароля
   Дата: 04.06.2026, 16:30:00 | Статус: Открыта

2. 🟡 Принтер не печатает чеки
   Дата: 03.06.2026, 10:15:00 | Статус: Открыта
```

## Умный поиск по FAQ

Бот понимает свободный текст. Просто напишите вопрос — бот найдёт ответ в базе FAQ.

**Примеры:**

| Вы пишете | Бот находит |
|-----------|------------|
| как добавить товар | FAQ: Как добавить новый товар |
| забыл пароль | FAQ: Как сменить пароль |
| принтер не работает | FAQ: Как распечатать чек |
| цена raos | FAQ: Сколько стоит RAOS |
| возврат товара | FAQ: Как сделать возврат |
| нет интернета | FAQ: Что при потере интернета |
| как нанять сотрудника | FAQ: Как добавить сотрудника |
| долг клиента | FAQ: Как оформить насию |

Если бот не нашёл ответ — предложит /faq, /ticket или /status.

После каждого ответа — кнопки:
- "Спасибо, помогло!" — если ответ помог
- "Не помогло — создать тикет" — если нужна помощь от человека

## Автоматический мониторинг (24/7)

Бот сам проверяет все сервисы каждые 3 минуты. Пользователю ничего делать не нужно.

### Как работает

1. Каждые 3 минуты бот отправляет HTTP запрос на 4 URL
2. Если сервис не отвечает 2 раза подряд (6 минут) — алерт
3. Алерт отправляется в Telegram группу поддержки и лично разработчикам
4. Когда сервис восстанавливается — сообщение "Восстановлен"

### Пример алерта при падении

```
🚨 СЕРВИС УПАЛ

🔴 Web Admin
URL: https://app.raos.uz
Ошибка: TIMEOUT — сервер не ответил за 10 секунд
Сбоев подряд: 2
Время: 15:36:00

🔍 Диагноз: Таймаут соединения
Причина: Сервис не отвечает — возможно крашнулся или Railway перезапускает

Как исправить:
1. railway service link <name> && railway logs --lines 50
2. Ищи: OOM, MODULE_NOT_FOUND, ENOENT
3. Если OOM: NODE_OPTIONS=--max-old-space-size=512
4. Redeploy: git commit --allow-empty && git push origin main
```

### Пример сообщения при восстановлении

```
✅ СЕРВИС ВОССТАНОВЛЕН

🟢 Web Admin
URL: https://app.raos.uz
Время ответа: 450ms
Время: 15:42:00
```

## Техническая информация

### Где находится код
```
apps/support-bot/
├── Dockerfile              # Docker образ для Railway
├── railway.toml            # Конфигурация Railway деплоя
├── package.json            # Зависимости (grammY, node-cron, dotenv)
├── src/
│   ├── main.ts             # Точка входа (с retry при конфликте polling)
│   ├── bot.ts              # Регистрация команд
│   ├── config.ts           # Переменные окружения
│   ├── handlers/
│   │   ├── start.handler.ts    # /start
│   │   ├── help.handler.ts     # /help
│   │   ├── faq.handler.ts      # /faq + inline keyboard
│   │   ├── status.handler.ts   # /status + диагностика
│   │   ├── ticket.handler.ts   # /ticket + /mytickets
│   │   └── message.handler.ts  # Свободный текст → FAQ поиск
│   ├── services/
│   │   ├── faq.service.ts      # Keyword matching по FAQ базе
│   │   └── monitor.service.ts  # HTTP health checks
│   ├── cron/
│   │   └── uptime.cron.ts      # Проверка каждые 3 минуты
│   └── data/
│       ├── faq-entries.ts       # 20 FAQ записей (рус + узб)
│       └── diagnostic-rules.ts  # 9 правил диагностики
```

### Railway сервис
- Имя: `support-bot`
- Деплой: автоматический при push в main
- Restart: ON_FAILURE (до 5 попыток)
- Dockerfile: `apps/support-bot/Dockerfile`

### Переменные окружения (Railway → Variables)
```
SUPPORT_BOT_TOKEN        # Токен бота от @BotFather
SUPPORT_CHANNEL_ID       # Chat ID группы поддержки (опционально)
ESCALATION_CHAT_IDS      # Chat ID разработчиков через запятую (опционально)
MONITOR_INTERVAL_MIN     # Интервал проверки в минутах (по умолчанию 3)
NODE_ENV                 # production
```

### Локальный запуск
```bash
# Добавь в .env:
SUPPORT_BOT_TOKEN=<токен от BotFather>

# Запуск:
pnpm --filter support-bot dev
```
